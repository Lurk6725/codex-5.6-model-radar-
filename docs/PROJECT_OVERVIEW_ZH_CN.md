# GPT‑5.6 Codex 模型雷达：中文项目说明

## 项目定位

本项目是一个独立、非商业、开源的 GPT‑5.6 Codex 模型长期监控与二次分析项目。它以 Codex Radar 公开展示的测试结果为基础，持续记录 Luna、Terra、Sol 各模型家族及不同推理强度的表现，并生成可复现的统计分析和使用建议。

本项目不是 OpenAI、Codex Radar 或 DeepSWE 的官方项目，也不代表这些组织的立场。

## 数据来源

主要数据来源为：

- Codex Radar 公开网页；
- Codex Radar 公开摘要；
- 人工核对的公开页面快照；
- 本仓库根据公开结果计算的衍生指标。

每个批次应尽量保留测试时间、批次 ID、模型与推理强度、通过数、逐题结果、Token、执行时间、估算费用、来源 URL 和数据精度说明。

项目不使用私有凭证，不规避访问限制，也不冒充 Codex Radar 官方镜像。未来如需私有 API、完整历史导出或受保护数据集再分发，将先与来源维护者沟通。

## 项目为什么存在

固定十题的单轮原始通过数容易被随机 Agent 轨迹影响，也无法区分简单题和困难题。本项目试图回答：

1. 哪个模型档位在多轮测试中最稳定？
2. 推理强度提高是否真的带来单调收益？
3. 相同订阅额度下，哪个档位提供更高的有效能力？
4. 哪些批次可能受后台配置、Harness、自动 Review 或服务状态影响？
5. 不同工作类型应该优先使用哪个模型？

## 难度加权

```text
原始难度_i = 1 / sqrt(历史通过率_i)
题目权重_i = 100 × 原始难度_i / sum(原始难度)
难度加权分 = sum(题目权重_i × 是否通过_i)
```

平方根用于限制最难单题对排行榜的支配程度。项目同时保留原始通过数，方便观察绝对成功数和任务难度结构。

## 主要统计指标

- **原始通过数**：十题中完成的数量；
- **难度加权分**：0–100；
- **加权 IQ**：难度加权分乘以 1.5；
- **额度效率**：难度加权分除以来源网站估算费用；
- **滚动中位数和均值**：观察长期中心趋势；
- **波动范围**：近期最好与最差表现；
- **Pareto 前沿**：没有其他配置同时更便宜且更强的模型档位；
- **异常修正趋势**：异常批次保留，但降低长期权重。

## 异常批次

异常批次不会被删除，因为它可能反映真实服务状态变化。默认异常信号包括：

- 全体通过数相较上一轮下降至少 15%；
- 同时总费用或总 Token 上升至少 30%；
- 多个互不相关的档位同时出现“更贵、更慢、成功更少”；
- 来源页面标记了 Harness、上下文、自动 Review 或后台配置变化。

## 订阅额度解释

面向 Codex 订阅用户时，本项目不将 API 的长上下文阶梯价格直接套用到订阅额度上。网站的美元等价费用用于相对比较，但不是精确账户账单。总 Token 和耗时用于判断 Agent 膨胀和交互延迟。

## 推荐原则

项目不会因为单轮测试结果就更换默认推荐。通常需要连续正常批次、最近五轮中位数变化、持续的 Pareto 优势、已记录的配置变化，或私人项目中的可复现实验。

当前推荐分为：

- 机械、低风险、允许重试；
- 日常开发默认；
- 高难、额度优先；
- 高难、最高能力优先；
- 当前不建议作为默认的档位。

## 当前限制

- 测试集仅有十道题；
- 每个任务通常只运行一次；
- Agent 轨迹具有随机性；
- 任务难度权重可能包含模型家族偏好；
- 公开摘要可能缺失逐题历史或部分重跑信息；
- 网站估算费用并非官方订阅扣款；
- 固定代码任务不能代表所有真实开发场景。
